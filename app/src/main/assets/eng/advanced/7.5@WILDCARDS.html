SQL WILDCARDS

SQL wildcards can be used when searching for data in a database.

The SQL Wildcards

SQL wildcards can substitute for one or more characters when searching for data in a database.

SQL wildcards must be used with the SQL LIKE operator.

With SQL, the following wildcards can be used:

Wildcard Description
% A substitute for zero or more characters
_ A substitute for exactly one character
[charlist] Any single character in charlist
[^charlist]

or

[!charlist]		 Any single character not in charlist

SQL Wildcard Examples

We have the following "Persons" table:

P_Id LastName FirstName Address City
1 Jameson John Streets 15 Sander
2 Smith Kate Green 68 Sander
3 Kristensen Olya Ski 2 Stavn

Using the % Wildcard

Now we want to select the persons living in a city that starts with "sa" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE City LIKE 'sa%'

The "%" sign can be used to define wildcards (missing letters in the pattern) both before and after the pattern.

The result-set will look like this:

P_Id LastName FirstName Address City
1 Jameson John Streets 15 Sander
2 Smith Kate Green 68 Sander

Next, we want to select the persons living in a city that contains the pattern "der" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE City LIKE '%der%'

The result-set will look like this:

P_Id LastName FirstName Address City
1 Jameson John Streets 15 Sander
2 Smith Kate Green 68 Sander

Using the _ Wildcard

Now we want to select the persons with a first name that starts with any character, followed by "ly" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE FirstName LIKE '_ly'

The result-set will look like this:

P_Id LastName FirstName Address City
3 Kristensen Olya Ski 2 Stavn

Next, we want to select the persons with a last name that starts with "J", followed by any character, followed by "mes", followed by any character, followed by "n" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE LastName LIKE 'J_mes_n'

The result-set will look like this:

P_Id LastName FirstName Address City
1 Jameson John Streets 15 Sander

Using the [charlist] Wildcard

Now we want to select the persons with a last name that starts with "b" or "k" or "s" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE LastName LIKE '[bks]%'

The result-set will look like this:

P_Id LastName FirstName Address City
2 Smith Kate Green 68 Sander
3 Kristensen Olya Ski 2 Stavn

Next, we want to select the persons with a last name that do not start with "b" or "k" or "s" from the "Persons" table.

We use the following SELECT statement:

SELECT * FROM Persons
WHERE LastName LIKE '[!bks]%'

The result-set will look like this:

P_Id LastName FirstName Address City
1 Jameson John Streets 15 Sander
